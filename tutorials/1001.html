2. ECS

Before diving into the engine let's first understand what ECS means:
ECS: for example in real life you have a phone or a PC this phone or PC is called an object even you are an object but some objects can move and some cannot so the objects that can't move are called static objects they have no extra components they just sit there till someone moves them right?
But now you as a human can move so of course you have components these components are things like the brain and nerves that help you walk think and react
ECS works the same way in a game engine: an object in the engine is just an empty object basically an empty space that does nothing until you add components to it
These components can make it move stay still or even act like superman it all depends on what you add inside that object and what it's supposed to do
For example if you want to make a static object you just need to give it components that make it visible because by default all objects in a game engine are invisible or empty
To make them visible you need two things: a "Mesh" and a "MeshRenderer" component
We will explain all components later for now you just need to know what a mesh is and how to get one:

3. Mesh

A mesh is basically the 3D shape or model of an object in your scene it's made up of points called vertices that connect together to form triangles and create the surface of your object for example a cube has 8 vertices connected by edges to make 6 faces that form the cube so the mesh is what gives your object its visible shape

Here is how to get a mesh in Sandmod Engine:
To get a mesh you need a ".fbx" file which you can either download online or create yourself in Blender or any 3D modeling software
After you get the ".fbx" file it will contain your 3D model inside you can import that ".fbx" file into Sandmod Engine we will learn exactly how to import files later on just focus for now on what a mesh actually means

Once you import the ".fbx" file into Sandmod Engine it will automatically break it down into pieces a ".mesh" file and a ".prefab" file
If your ".fbx" contains more than one object like two cubes for example the engine will create two separate ".mesh" files one for each cube
The ".prefab" file however is different it doesn’t separate the file it keeps everything together exactly as it was in your 3D software it remembers the positions rotations and scales of all objects so it recreates the full setup

We will talk more about prefabs in a moment

4. Prefab

Now let's talk about what a "Prefab" is:
A prefab is basically a saved version of an object or a group of objects that keeps everything about them like their position rotation scale and the components attached to them
Think of it like a blueprint once you make a prefab you can use it anywhere in your project without needing to rebuild the object from scratch every time

For example:
Let's say you made a car in your scene that has wheels lights and a body you can turn that car into a prefab and then place as many copies of it as you want in your game world every copy will have the same setup as the original one
If you update the original prefab like changing the car color or adding a new component all other copies will also update automatically because they are all linked to that prefab

So in short:
A "Prefab" is a reusable object template that stores all the data of your object and helps you keep your scene organized and easy to manage

5. ECS Part Two

After knowing what a "Mesh" is and what an object means in a game engine let's talk about how to make the object visible:
By default objects in a game engine are just empty spaces they have no shape and nothing to render or show on the screen so to make them visible we need to add components that bring them to life

After we get the ".mesh" file we need to add a "MeshRenderer" component
Below is the full explanation of what a "MeshRenderer" is:

The "MeshRenderer" component is what takes your mesh and actually draws it on the screen it tells the engine how to display that mesh using materials and lighting without the MeshRenderer your mesh would still exist but it would be invisible because nothing is telling the engine to render it

Think of it like this:
The "Mesh" is the shape of your object and the "MeshRenderer" is the thing that shines light on it so you can see it
The MeshRenderer connects your mesh to materials so it knows what color texture or shader to use when displaying it in the scene

So in short:
A "Mesh" defines the shape
A "MeshRenderer" makes it visible and applies materials to it

After displaying our first object in the engine we still haven't actually created it inside the engine yet but we will soon before we do that we need to understand one more important thing which is the default component that every object has in a game engine and that is the "Transform" component

Any object that exists in a game engine must have a "Transform" component this component defines three main things about the object: its position its rotation and its scale
Without these three values the object would not exist in the 3D world it wouldn’t have a place to be shown or even a direction to face so if an object doesn’t have a transform component it’s not really a game object at all

Below is a full explanation of what the "Transform" component does:

The "Transform" component is what gives every object its place in the 3D world
It tells the engine where the object is how it is rotated and how big or small it is

Here is what each part of the transform does:

"Position": this decides where the object is located in the scene for example if you move the position on the X axis the object will move left or right if you move it on the Y axis it will move up or down and if you move it on the Z axis it will move forward or backward

"Rotation": this controls the direction the object is facing if you rotate on the X axis it will tilt forward or backward rotating on the Y axis will turn it left or right and rotating on the Z axis will spin it sideways

"Scale": this changes the size of the object if you increase the scale it becomes bigger if you decrease it it becomes smaller you can also change each axis separately to stretch or shrink the object in specific directions

So the transform is like the foundation of every object it defines where the object is how it is turned and how large it appears without it the engine would have no idea how or where to display your object
